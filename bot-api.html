Bot
API

Your companion is driven by a JavaScript function you write. Each game tick it receives the current state and must return an action. This page walks through the outputs, then the inputs, then builds up to full working examples.

The compute function

Define a global function named exactly compute. The game calls it once per tick (up to 60 fps). Return an action object every call. 

function compute({ now, me, companion, players }) {

  // decide what your companion does this tick

  return {
    turn:     0,      // rotate: -1 (left)  →  1 (right)
    throttle: 0,      // speed:   0 (stop)  →  1 (full)
    shoot:    false,  // fire a bullet
  };
}

The function runs inside a sandboxed Function() constructor — no access to document, window, or the network. Module-level variables persist between ticks, so you can track state (cooldowns, waypoints, phase flags). If the function throws or takes longer than 6 ms, that tick is skipped and a warning is logged to the console.

Outputs — return value

Return a plain object each tick. Omitted fields default to 0 / false. Values outside the valid range are clamped automatically.

Field Type / Range Description
turn number · −1 → 1 Rotation speed. −1 = full left, 1 = full right, 0 = no rotation. Turn speed is 5.5 rad/s at full value.
throttle number · 0 → 1 Forward speed. 0 = stop, 1 = full speed (220 px/s). The companion always moves in the direction it faces.
shoot boolean Set to true to fire. The server enforces a 280 ms cooldown — returning true every tick fires as fast as the limit allows.
aimAngle (optional) number · radians Independent aim direction for shooting. When omitted, bullets fire along the companion's current heading.
World reference

World dimensions: 900 × 540 px. Angles follow the canvas convention: 0 = right (+x), Math.PI / 2 = down (+y), Math.PI = left, −Math.PI / 2 = up. Use Math.atan2(dy, dx) to compute the angle from one point to another.

The interactive demo below lets you set turn and throttle directly and fire with the shoot button. The heading ray shows the companion's facing direction.

Inputs — the context object

The argument passed to compute contains four fields. Enemies are only present in players when your human or companion has line-of-sight to them within 320 px.

now number · ms

A high-resolution timestamp from performance.now(), in milliseconds. Use it to implement cooldowns or timed sequences without relying on tick counters. 

let lastFiredAt = 0;

function compute({ now }) {
  const FIRE_EVERY_MS = 600;

  const fire = now - lastFiredAt > FIRE_EVERY_MS;
  if (fire) lastFiredAt = now;

  return { turn: 0, throttle: 0, shoot: fire };
}
me object · human player

Your human character — the one you control with the keyboard or gamepad. Use it to make the companion follow, protect, or react to your position.

me.id — unique player ID string
me.x / me.y — world position in pixels (world is 900 × 540)
me.angle — facing direction in radians (0 = right, π/2 = down)
me.alive — false when eliminated

companion object · the bot you control

The companion character your script drives. Its ID always ends in :c (e.g. "p1:c"). Same shape as me.

companion.id — player ID (always ends in :c)
companion.x / companion.y — world position
companion.angle — current facing angle in radians
companion.alive — false when eliminated

players Map<string, player>

A Map of all currently visible players, keyed by player ID. Your own team is always present. Enemies appear only when your human or companion has line-of-sight to them within 320 px.

Each entry has: x, y, angle, alive, kind ("human" or "companion").

Team detection: a companion ID is the owner's base ID plus :c. Strip the suffix from both IDs and compare to identify teammates. 

function sameTeam(a, b) {
  return a.replace(/:c$/, '') === b.replace(/:c$/, '');
}

Demo — Escort

The simplest useful behaviour: follow your human teammate. Using me and companion together, compute the angle and distance to the leader, then steer toward them. The minDist threshold stops the companion from crowding you. 

// Escort — follow the human player, keep minimum distance.

function compute({ me, companion }) {
  if (!me || !me.alive) return { turn: 0, throttle: 0, shoot: false };

  const dx = me.x - companion.x;
  const dy = me.y - companion.y;
  const d  = Math.sqrt(dx * dx + dy * dy);

  if (d < 80) return { turn: 0, throttle: 0, shoot: false };

  const angle = Math.atan2(dy, dx);
  const delta = normAngle(angle - companion.angle);

  return {
    turn:     clamp(delta * 1.5, -1, 1),
    throttle: clamp(d / 200, 0.3, 1),
    shoot:    false,
  };
}

function normAngle(a) {
  while (a >  Math.PI) a -= Math.PI * 2;
  while (a < -Math.PI) a += Math.PI * 2;
  return a;
}

function clamp(v, lo, hi) { return Math.max(lo, Math.min(hi, v)); }
// interactive · escort · follow leader
minDist 80 px
turnMult 1.5

The blue tank is your human (patrolling a figure-8). The green tank is the companion. Drag minDist to change how close the companion gets before stopping. Drag turnMult to change how aggressively it steers.

Demo — Escort + Combat

When an enemy appears in players, switch to combat mode: close to an ideal range, hold position, and fire. When the enemy leaves line-of-sight, fall back to escorting the human. This two-mode pattern is the foundation of most effective companion strategies.

// interactive · escort + combat · ranger bot
closeRange 110 px
idealRange 190 px
shootRange 260 px

Blue = human (patrolling). Green = companion (your bot). Red = enemy. The companion switches between ESCORT and COMBAT as the enemy enters and exits the 320 px line-of-sight range. Adjust the three range sliders and watch how the companion repositions. 

// ── Ranger bot ────────────────────────────────────────────────────────────────
// - Visible enemy  →  orbit at ideal range and shoot
// - No enemy       →  escort the human player
// ─────────────────────────────────────────────────────────────────────────────

function compute({ me, companion, players }) {

  // 1. Find nearest visible enemy
  let target = null, targetDist = Infinity;
  for (const [id, p] of players) {
    if (!p.alive || id === companion.id || sameTeam(companion.id, id)) continue;
    const d = dist(companion, p);
    if (d < targetDist) { targetDist = d; target = p; }
  }

  // 2. Combat mode — enemy is visible
  if (target) {
    const IDEAL = 190;   // px — comfortable fighting distance
    const CLOSE = 110;   // px — too close, back up
    const SHOOT = 260;   // px — max firing range

    const toTarget = Math.atan2(target.y - companion.y, target.x - companion.x);
    const delta    = normAngle(toTarget - companion.angle);
    const throttle = targetDist < CLOSE ? -0.6
                   : targetDist > IDEAL ?  0.7
                   :                       0.1;

    return {
      turn:     clamp(delta * 1.6, -1, 1),
      throttle: throttle,
      shoot:    targetDist < SHOOT,
    };
  }

  // 3. Escort mode — no enemy visible, follow the human
  if (!me || !me.alive) return { turn: 0, throttle: 0, shoot: false };

  const leaderDist = dist(companion, me);
  if (leaderDist < 80) return { turn: 0, throttle: 0, shoot: false };

  const toLeader = Math.atan2(me.y - companion.y, me.x - companion.x);
  const delta    = normAngle(toLeader - companion.angle);

  return {
    turn:     clamp(delta * 1.5, -1, 1),
    throttle: clamp(leaderDist / 180, 0.3, 0.9),
    shoot:    false,
  };
}

function dist(a, b) {
  return Math.sqrt((a.x - b.x) ** 2 + (a.y - b.y) ** 2);
}

function sameTeam(a, b) {
  return a.replace(/:c$/, '') === b.replace(/:c$/, '');
}

function normAngle(a) {
  while (a >  Math.PI) a -= Math.PI * 2;
  while (a < -Math.PI) a += Math.PI * 2;
  return a;
}

function clamp(v, lo, hi) { return Math.max(lo, Math.min(hi, v)); }

Paste this into the bot editor under Options → Bots → Add, give it a name, and save. Select it in Practice to test it against the built-in AI before using it in a real match.

Demo — Exploration

Enemies behind walls are invisible — they're outside line-of-sight. Your bot can't see or target them. Bots often need to explore: navigate toward a known area while routing around obstacles.

The demo shows a three-phase state machine: approach (head toward the target), scan (slide along the wall to find the gap), and cross (pass through). Drag the slider to move the gap and watch the bot adapt its route in real time.

// interactive · wall navigation · exploration
gap Y 270 px 
// Exploration — navigate through a known wall passage via a phase state machine.
// Wall positions are hardcoded here; in practice, derive them from the map layout
// or estimate from collision heuristics (noticing when movement is blocked).

let phase = 'approach';
const WALL_X = 480;          // x position of the wall (world units)
const GAP_Y  = 270;          // estimated centre of the gap
const TARGET = { x: 770, y: 270 };   // destination on the far side

function compute({ companion }) {
  switch (phase) {

    case 'approach':
      // Head straight for the target; when near the wall, check alignment with gap
      if (companion.x > WALL_X - 58)
        phase = Math.abs(companion.y - GAP_Y) < 44 ? 'cross' : 'scan';
      return steerTo(companion, TARGET.x, TARGET.y);

    case 'scan':
      // Slide along the wall face toward the gap centre
      if (Math.abs(companion.y - GAP_Y) < 32) phase = 'cross';
      return steerTo(companion, WALL_X - 46, GAP_Y);

    case 'cross':
      // Drive through the gap
      if (companion.x > WALL_X + 60) phase = 'done';
      return steerTo(companion, WALL_X + 95, GAP_Y);

    default:
      return steerTo(companion, TARGET.x, TARGET.y);
  }
}

function steerTo(bot, tx, ty) {
  const angle = Math.atan2(ty - bot.y, tx - bot.x);
  const delta = normAngle(angle - bot.angle);
  return { turn: clamp(delta * 2.5, -1, 1), throttle: 0.75, shoot: false };
}

function normAngle(a) {
  while (a >  Math.PI) a -= Math.PI * 2;
  while (a < -Math.PI) a += Math.PI * 2;
  return a;
}

function clamp(v, lo, hi) { return Math.max(lo, Math.min(hi, v)); }

The players map only contains player state — walls are not exposed directly by the API. Effective navigation requires either hardcoding known map geometry, inferring obstacles from movement blocking, or using a trained ML model that learns the layout implicitly.
